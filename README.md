# worker-backend

Production-grade REST API built on **Cloudflare Workers** using Hono, Drizzle ORM, Better Auth, and D1 вЂ” with full local, staging, and production environment support.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Runtime | [Cloudflare Workers](https://workers.cloudflare.com/) (`workerd`) |
| Framework | [Hono](https://hono.dev/) + [`@hono/zod-openapi`](https://github.com/honojs/middleware/tree/main/packages/zod-openapi) |
| Database | [Cloudflare D1](https://developers.cloudflare.com/d1/) (SQLite at the edge) |
| ORM | [Drizzle ORM](https://orm.drizzle.team/) |
| Auth | [Better Auth](https://www.better-auth.com/) |
| Validation | [Zod](https://zod.dev/) |
| Linting / Formatting | [Biome](https://biomejs.dev/) |
| Testing | [Vitest](https://vitest.dev/) + [`@cloudflare/vitest-pool-workers`](https://www.npmjs.com/package/@cloudflare/vitest-pool-workers) |
| Package Manager | [pnpm](https://pnpm.io/) |
| Deployment | [Wrangler](https://developers.cloudflare.com/workers/wrangler/) |

## Cloudflare Workers Setup

### Environment Variables

**Important:** In Cloudflare Workers, `process.env` does NOT work. Environment variables are accessed through the `env` parameter passed to your worker handler.

#### For Development (Local)
Create a `.env` file with:
```bash
# Database URL for drizzle-kit (build time only)
DATABASE_URL=https://your-account-id.cloudflare.com/d1/databases/your-database-id

# Secrets (set via wrangler)
AUTH_SECRET=your-32-character-secret-key
AUTH_REDIRECT_URL=http://localhost:8787/auth/callback
```

#### For Production
Set secrets using Wrangler:
```bash
wrangler secret put AUTH_SECRET --env production
wrangler secret put AUTH_REDIRECT_URL --env production
```

### Database Bindings

Update `wrangler.json` with your actual resource IDs:
```json
{
  "d1_databases": [
    {
      "binding": "DB",
      "database_name": "aiyeen-db",
      "database_id": "your-actual-database-id"
    }
  ]
}
```

### Accessing Environment in Code

```typescript
// вњ… Correct: Use env parameter
export default {
  fetch: (request, env, ctx) => {
    // Access D1 database
    const db = createDb(env);

    // Access secrets
    const authSecret = env.AUTH_SECRET;

    // Access KV namespace
    const cache = env.SESSION_CACHE;
  }
}

// вќЊ Wrong: process.env doesn't work in Workers
const secret = process.env.AUTH_SECRET; // undefined
```

## Environment Setup

### 1. Copy and fill secrets

```bash
cp .dev.vars.template .dev.vars
```

`.dev.vars` is Wrangler's local secrets file (equivalent to `.env` for Workers). It is **gitignored**.

### 2. Generate Cloudflare binding types

```bash
pnpm types:gen
```

This runs `wrangler types` and generates `worker-configuration.d.ts` вЂ” giving you type-safe access to `env.DB`, `env.BETTER_AUTH_SECRET`, etc.

## Quality Assurance Tools

### Lefthook (Git Hooks)

Git hooks are configured via `lefthook.yml` and run automatically on commits:

- **pre-commit**: Runs linting, formatting, and type checking
- **commit-msg**: Validates conventional commit messages
- **pre-push**: Runs full test suite

```bash
pnpm lefthook install  # Install hooks (runs automatically via prepare script)
pnpm lefthook run pre-commit  # Manual pre-commit check
```

### Commitlint (Conventional Commits)

Enforces conventional commit format using `.commitlintrc.json`:

```bash
# вњ… Valid commits
git commit -m "feat: add user authentication"
git commit -m "fix: resolve database connection issue"
git commit -m "docs: update API documentation"

# вќЊ Invalid commits
git commit -m "added auth"  # Missing type
git commit -m "FIX: bug"    # Wrong case
```

### Knip (Unused Dependencies)

Detects unused dependencies and exports using `knip.json`:

```bash
pnpm knip              # Check for unused dependencies
pnpm knip:fix          # Remove unused dependencies automatically
```

## Logging

`pino` is **not compatible** with the Cloudflare Workers (`workerd`) runtime вЂ” it uses Node.js stream APIs.

This project uses a lightweight custom structured logger in `src/utils/logger.ts` that writes JSON to `console.*`. Cloudflare captures these logs in:

- **Local:** `wrangler tail` in your terminal
- **Staging/Production:** Cloudflare Workers Logs dashboard
- **Advanced:** Cloudflare Logpush в†’ Axiom, Datadog, S3, etc.

```typescript
import { logger } from '@/utils'

logger.info('User created', { userId: '123' })
logger.error('DB query failed', { table: 'users', error: err.message })
```

## Key Decisions

| Decision | Reason |
|---|---|
| Single `wrangler.json` with `env` | Cloudflare-recommended pattern вЂ” no config duplication |
| Biome over ESLint + Prettier | Single tool, faster, consistent rules |
| `@hono/zod-openapi` only (no `@hono/zod-validator`) | `zod-openapi` includes validation вЂ” installing both causes conflicts |
| Custom logger over `pino` | `pino` is Node.js-only вЂ” incompatible with `workerd` |
| `@cloudflare/vitest-pool-workers` | Tests run inside real `workerd` runtime вЂ” matches production exactly |
| No `miniflare` in deps | Embedded inside Wrangler v3+ вЂ” never install separately |
| No `tsx` in devDeps | Wrangler bundles via `esbuild` internally вЂ” `tsx` is unused |
| `tests/` at root (not co-located) | Clean separation; integration tests need separate `workerd` config |
| Lefthook for Git hooks | Cross-platform Git hooks with YAML config |
| Commitlint for conventional commits | Enforces consistent commit messages |
| Knip for unused dependency detection | Prevents bloated bundle sizes |
| `.dev.vars.template` | Template for local secrets setup |
| `fixtures/` in tests | Centralized test data management |
| `AGENTS.md` + `CLAUDE.md` | AI assistant guidelines for consistent development |
| `cf-workers-best-practices.md` | Cloudflare Workers production guidelines |
